## 读书心得
给出了可运行的实例与每个阶段演进的原因。

密码存储：加盐+慢哈希。

加盐：在密码字符串前或后添加一个随机字符串，再对字符串进行哈希

慢哈希：是 hash 字符串的一种算法类型，主流的用户密码存储推荐方案是 Bcrypt。“慢” 针对的是生成时，CPU 和内存开销都更大。

慢哈希算法在给攻击者带来时间和资源成本等困难的同时，也给服务端正常的身份认证带来一定的性能开销，不过大多数开发者认为这种设计取舍是值得的。
